# senior-dev-workflow

> "Choose the method that prevents circumstances from controlling the development."

## Overview

This skill is specifically engineered for working within **legacy projects**—systems with long histories, a vast trail of experiments, and a codebase shaped by multiple developers of varying skill levels over time. While these projects are functional and have ambitious expansion plans, they often contain hidden complexities, "dark matter" dependencies, and outdated documentation.

The `senior-dev-workflow` is designed to bring order to this chaos. It transforms the development process from a reactive "guess-and-check" approach into a proactive, disciplined engineering pipeline.

## What This Skill Provides

This workflow simplifies and organizes the development process by introducing a rigorous, reasonable sequence of actions:

1. **Contextual Dominance:** Instead of diving into code, it forces the agent to map the problem space and uncover non-obvious dependencies first.
2. **Knowledge Persistence:** It uses a Git-indexed Knowledge Base to ensure that insights gained today are not lost tomorrow and are not duplicated across tasks.
3. **Architectural Integrity:** It elevates architecture from a side-thought to a global constant, ensuring that new features align with the long-term vision.
4. **Disciplined Error Recovery:** It treats bugs not as nuisances, but as data points, requiring a "post-mortem" analysis to prevent the same mistake from recurring.
5. **Self-Documenting Evolution:** It prioritizes code clarity over comments, ensuring the system remains maintainable as it scales.

By implementing this skill, an AI agent ceases to be a mere code generator and becomes a Senior Engineer capable of navigating, stabilizing, and evolving complex legacy systems without being overwhelmed by their history.

## Installation & Usage

To activate this workflow, add the `senior-dev-workflow` skill to your agent's configuration. The agent will automatically adopt the specified roles, git-flow, and knowledge management strategies described in the `SKILL.md` file.
